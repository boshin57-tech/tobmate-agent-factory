# Writing Style

- Preserve hierarchical numbering exactly.
- Use formal Korean specification prose.
- Keep English technical terms in parentheses where useful.
- Each section should include: Purpose, Architecture or Rule, State/Flow, Validation, Failure handling, Audit, Invariants when applicable.
- Avoid repeating prior constitutional conclusions.
- Prefer implementation-ready object models and state transitions.
- Never invent cross-references unless the referenced section exists in the source index.
- Mark unresolved references as [REFERENCE_REQUIRED].
- Do not claim code is implemented unless code files exist.
